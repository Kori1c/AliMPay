# 发布说明

## 当前版本收尾说明

本次版本主要完成了三类工作：

- 补齐 Docker 启动自检能力，容器启动后会自动执行轻量自检
- 将系统健康状态接入后台，异常时可以在 Web 端直接看到提示
- 修正线上二维码加载与轮询相关问题，并同步优化部署文档

当前版本已经完成：

- 本地 Docker 启动验证
- 容器健康检查验证
- 关键 PHP 文件语法校验
- 线上环境代码同步与可用性验证
- GitHub 主分支同步

当前建议作为可发布版本使用，但正式开放给更多人之前，仍建议先完成一轮面向部署者的回归测试：

- 下单
- 支付
- 异步回调
- 订单轮询
- 经营码偏移
- 备份恢复

## 本次适合对外说明的更新点

可直接用于 GitHub Release 或项目更新公告：

1. 新增启动自检机制，服务启动后会自动检查关键运行条件。
2. 后台新增系统健康状态提示，部署异常更容易发现。
3. 修复生产环境二维码资源加载异常。
4. 优化订单轮询相关展示与管理交互。
5. README 已改为更适合部署者阅读的版本。
6. 项目已支持通过 GitHub Actions 自动构建并发布 Docker 镜像。

## GitHub Release 文案

### 标题建议

`v1.0.5 - 启动自检与系统健康状态增强`

### 正文建议

本次更新重点增强了部署稳定性与问题可见性，适合准备正式对外使用或公开部署的场景。

更新内容：

- 新增启动自检机制，容器启动后会自动检查关键运行条件
- 后台新增系统健康状态提示，部署异常更容易在 Web 端发现
- 修复生产环境二维码资源加载异常
- 优化订单轮询相关展示与管理交互
- 更新部署文档，降低首次部署和排查成本

升级建议：

- 拉取新版本镜像或最新代码后重启容器
- 登录后台确认系统健康状态是否正常
- 建议手动创建一笔测试订单，确认下单、支付、回调、轮询链路可用

注意：

- 请勿提交或公开你的真实支付宝配置、商户密钥、订单数据库和经营码图片
- 首次部署完成后请立即修改后台默认密码
